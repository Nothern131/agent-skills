# Agent Skills — 项目介绍

## 项目名称

**Agent Skills：让任何 AI 编码助手达到 GPT 级别输出质量的工程框架**

## 一句话描述

通过结构化工程纪律（而非模型能力本身），让国产 AI 模型的代码生成、深度推理、任务完成和项目落地能力达到 GPT-4/Claude 水平。

## 项目背景

### 现状

国产 AI 大模型（通义千问、智谱 GLM、DeepSeek、百川、豆包等）在单轮代码生成上已经非常接近 GPT-4 和 Claude 的水平。但在实际工程场景中，差距依然明显：

| 能力维度 | GPT-4/Claude | 国产模型 | 差距原因 |
|---------|-------------|---------|---------|
| 单轮代码生成 | 95分 | 85分 | 模型能力差距小 |
| 多步骤复杂推理 | 90分 | 60分 | 上下文丢失、步骤跳跃 |
| 指令遵循 | 95分 | 70分 | 忽略次要约束 |
| 自我修正 | 85分 | 45分 | 错误恢复能力弱 |
| 长上下文维护 | 90分 | 55分 | 细节遗漏 |
| 项目级落地 | 80分 | 40分 | 缺乏工程纪律 |

### 核心洞察

**差距不在模型能力，在工程过程。**

如果国产模型的原始能力是 GPT 的 80%，但通过工程纪律乘以 1.5 倍的放大系数，最终输出质量就是 120%——超越 GPT。

```
Quality = Model_Raw_Capability × Engineering_Multiplier
```

## 核心价值

### 1. 过程弥补能力（model-upgrade）

四层防御体系，每层解决一个核心差距：

| 层级 | 解决的差距 | 机制 |
|------|-----------|------|
| 铁律规则 | 指令遵循差 | 15条不可违反的工程纪律 |
| 项目感知 | 长上下文丢失 | PROJECT_CONTEXT.md 全局视野 |
| 任务拆解 | 多步骤推理弱 | 大任务→小步骤+检查点 |
| 测试修正循环 | 自我修正差 | 生成→验证→修正→验证循环 |

### 2. 代码质量保障（codeguard）

80 条检测规则覆盖 8 个维度，0-100 评分，优先级排序的修复建议：

- 错误处理（ERR001-010）
- 安全漏洞（PERM001-010）
- 数据库保护（DB001-010）
- 问题诊断（DIAG001-010）
- 韧性设计（RES001-010）
- 内存安全（MEM001-010）— C/C++ 专项
- 并发安全（CON001-CON010）— 含 GDScript
- 资源管理（RES_M001-010）— 含 GDScript 节点生命周期

### 3. 需求精准理解（clarifying-questions）

在动手之前先问对问题，避免"做对了错误的事"。

## 创新点

### 创新 1：工程乘数效应

业界普遍认为"模型能力决定输出质量"。本项目提出并验证了"工程过程可以放大模型能力"的观点——通过结构化纪律，80% 能力的模型可以产出 120% 质量的代码。

### 创新 2：全语言覆盖的代码审查

现有代码审查工具要么只支持特定语言（如 SonarQube 对 C/C++ 支持有限），要么只做语法检查不做语义分析。codeguard 的 80 条规则覆盖 Python、JS/TS、C/C++、GDScript、Go 五种语言，每条规则都有违规模式+修复示例。

### 创新 3：国产模型专项优化

model-upgrade 不是通用的"最佳实践"集合，而是针对国产模型的具体弱点（上下文丢失、指令跳跃、自修正弱）设计的精准补偿方案。每条规则都有明确的"解决什么差距"的映射。

### 创新 4：Skill 标准化

三个 Skill 都遵循 Agent Skills 规范（YAML frontmatter + SKILL.md + references/），可以在 Claude Code、Trae、Cursor、Windsurf、Cline、Aider、Continue、GitHub Copilot 等 10+ 个 AI 编码工具中使用。

## 适用场景

- **企业开发团队**：使用国产模型降低成本，但需要 GPT 级别的代码质量
- **个人开发者**：使用免费/低价国产模型，但不想牺牲代码质量
- **游戏开发者**：GDScript 专项规则（节点生命周期、deferred call 安全）
- **C/C++ 项目**：内存安全专项规则（缓冲区溢出、use-after-free、RAII）
- **AI 工具开发者**：为自己的 AI 编码工具集成质量保障能力

## 项目规模

- **3 个 Skill 模块**：codeguard（80规则）、model-upgrade（4层防御）、clarifying-questions
- **5 种语言覆盖**：Python、JS/TS、C/C++、GDScript、Go
- **8 个代码审查维度**：错误处理、安全、数据库、诊断、韧性、内存、并发、资源
- **10+ 个 AI 工具兼容**：Claude Code、Trae、Cursor、Windsurf、Cline 等
- **9 个 Godot 编辑器插件**：碰撞生成、摄像机特效、调试叠加、对话编辑器等
